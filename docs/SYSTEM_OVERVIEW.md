# System Overview

FireRoute KU is organized around one operational loop:
1. incident intake
2. explainable triage and dispatch confidence
3. direct or hydrant-aware routing
4. responder assignment
5. governance and evidence export

## Core data layers
- pilot graph (nodes + edges)
- hydrant status layer
- environmental sensor layer
- responder fleet state
- incident/governance records

## Core operational outputs
- direct route ETA
- hydrant-aware chain ETA
- explainable risk score and breakdown
- audit log timeline
- maintenance tickets
- evidence-pack ZIP
